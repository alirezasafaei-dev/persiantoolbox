## P1 — تکمیل فیچرهای نیمه‌کاره

### 1. فعال‌سازی احراز هویت (auth)

- **فایل‌ها**: `lib/features/availability.ts`, `lib/server/auth.ts`, `lib/server/sessions.ts`, `app/api/auth/*`
- **وضعیت فعلی**: فلگ `auth: defaultEnabled: false`. API لاگین وجود دارد ولی غیرفعال است.
- **کار لازم**:
  - بررسی صحت API لاگین/لاگاوت/رجیستر
  - اطمینان از صحت session management
  - فعال‌سازی فلگ `auth`
  - تست جریان لاگین کامل
- **خروجی**: کاربران بتوانند ثبت‌نام و ورود کنند

### 2. فعال‌سازی حساب کاربری (account)

- **فایل‌ها**: `app/account/page.tsx`, `lib/features/availability.ts`
- **وضعیت فعلی**: فلگ `account: defaultEnabled: false`
- **کار لازم**:
  - بررسی صحت صفحه account
  - اتصال به سیستم احراز هویت
  - نمایش اطلاعات کاربر و تاریخچه
  - فعال‌سازی فلگ `account`
- **خروجی**: صفحه حساب کاربری فعال و کاربردی

### 3. فعال‌سازی اشتراک‌ها (subscription)

- **فایل‌ها**: `app/(tools)/subscription/page.tsx`, `app/api/subscription/*`, `lib/subscriptions/`
- **وضعیت فعلی**: فلگ `subscription: defaultEnabled: false`. صفحه "به‌زودی" نشان می‌دهد.
- **کار لازم**:
  - بررسی API اشتراک (checkout, confirm, status, webhook)
  - اتصال به سیستم پرداخت
  - فعال‌سازی فلگ `subscription`
  - حذف بنر "به‌زودی"
- **خروجی**: سیستم اشتراک فعال

### 4. فعال‌سازی طرح‌های اشتراک (plans)

- **فایل‌ها**: `app/plans/page.tsx`, `lib/features/availability.ts`, `lib/subscriptionPlans.ts`
- **وضعیت فعلی**: فلگ `plans: defaultEnabled: false`
- **کار لازم**:
  - بررسی صحت صفحه plans
  - اتصال به داده‌های واقعی plan
  - فعال‌سازی فلگ `plans`
- **خروجی**: صفحه نمایش طرح‌های اشتراک فعال

### 5. فعال‌سازی پرداخت (checkout)

- **فایل‌ها**: `app/checkout/[id]/page.tsx`, `app/checkout/layout.tsx`, `app/api/subscription/checkout/route.ts`
- **وضعیت فعلی**: فلگ `checkout: defaultEnabled: false`
- **کار لازم**:
  - بررسی صحت جریان پرداخت
  - اتصال به درگاه پرداخت
  - فعال‌سازی فلگ `checkout`
- **خروجی**: جریان پرداخت کامل

### 6. فعال‌سازی پریمیوم (premium)

- **فایل‌ها**: `app/(tools)/premium/page.tsx`, `lib/premium/premium-features.ts`
- **وضعیت فعلی**: صفحه "به‌زودی" نشان می‌دهد
- **کار لازم**:
  - بررسی منطق premium features
  - اتصال به سیستم اشتراک
  - حذف بنر "به‌زودی"
  - فعال‌سازی دکمه‌ها
- **خروجی**: صفحه پریمیوم فعال

### 7. رمزگذاری PDF (encrypt-pdf)

- **فایل‌ها**: `features/pdf-tools/security/encrypt-pdf.tsx`
- **وضعیت فعلی**: stub با پیام "پشتیبانی نمی‌شود"
- **کار لازم**:
  - بررسی کتابخانه‌های رمزگذاری PDF سازگار با مرورگر
  - پیاده‌سازی رمزگذاری با pdf-lib یا جایگزین
  - یا حذف ابزار از لیست و ریدایرکت
- **خروجی**: ابزار رمزگذاری PDF کاربردی یا حذف شده

### 8. فعال‌سازی پنل ادمین (admin)

- **فایل‌ها**: `app/admin/site-settings/page.tsx`, `app/admin/monetization/page.tsx`
- **وضعیت فعلی**: فلگ‌ها `admin-site-settings` و `admin-monetization` غیرفعال
- **کار لازم**:
  - بررسی صحت پنل‌های ادمین
  - اطمینان از دسترسی امن
  - فعال‌سازی فلگ‌ها
- **خروجی**: پنل مدیریت فعال

## P2 — بهبود کیفیت و عملکرد

### 9. تکمیل اپلیکیشن موبایل

- **فایل‌ها**: `mobile-app/src/screens/*`
- **وضعیت فعلی**: ۵ صفحه وجود دارد ولی فقط ۳ ابزار در ToolDetailScreen پیکربندی شده
- **کار لازم**:
  - اضافه کردن پیکربندی برای تمام ۳۴+ ابزار
  - تست روی iOS/Android
  - ساخت و تست اپ
- **خروجی**: اپ موبایل کامل و قابل انتشار

### 10. فعال‌سازی تبلیغات (ads)

- **فایل‌ها**: `app/ads/page.tsx`, `shared/ui/AdSlot.tsx`, `lib/features/availability.ts`
- **وضعیت فعلی**: فلگ `ads: defaultEnabled: false`
- **کار لازم**:
  - بررسی سیستم تبلیغات محلی
  - اطمینان از حریم خصوصی
  - فعال‌سازی فلگ
- **خروجی**: سیستم تبلیغات شفاف فعال

### 11. فعال‌سازی صفحه توسعه‌دهندگان (developers)

- **فایل‌ها**: `app/developers/page.tsx`, `app/developers/api/page.tsx`
- **وضعیت فعلی**: فلگ `developers: defaultEnabled: false`
- **کار لازم**:
  - بررسی صحت ابزارهای توسعه‌دهندگان (Base64, JSON Formatter)
  - فعال‌سازی فلگ
- **خروجی**: صفحه توسعه‌دهندگان فعال

### 12. بهینه‌سازی عملکرد

- **وضعیت**: فشرده‌سازی تصویر موازی شده (concurrency=3)
- **کار لازم**:
  - بهینه‌سازی bundle size framer-motion
  - بررسی Lighthouse scores
  - بهینه‌سازی بارگذاری فونت
- **خروجی**: Lighthouse score بالای ۹۰

### 13. بهبود تست‌ها

- **وضعیت**: ۳۲۷ تست واحد + ۱۱ تست بصری
- **کار لازم**:
  - اضافه کردن تست برای API route‌ها
  - اضافه کردن تست برای فیچرهای جدید
  - افزایش coverage به بالای ۹۰٪
- **خروجی**: پوشش تست جامع

### 14. یکپارچه‌سازی agent/automation فایل‌ها

- **فایل‌ها**: ۱۸ فایل در `lib/` (bug-triage, code-review, etc.)
- **وضعیت فعلی**: برخی placeholder data دارند
- **کار لازم**:
  - بررسی و تکمیل هر فایل
  - یا حذف فایل‌های غیرضروری
- **خروجی**: کد تمیز بدون placeholder
