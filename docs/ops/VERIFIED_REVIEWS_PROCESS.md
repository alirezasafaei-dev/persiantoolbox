# Verified Reviews Collection Process — PersianToolbox

**Status:** ACTIVE process (docs only until real testimonials exist)  
**Date:** 2026-07-09  
**Rule:** Never publish fabricated user quotes.

## Why

Trust score is blocked by placeholder social proof. Honest process > fake quotes.

## Sources allowed

1. Telegram channel support DMs with **explicit permission** to publish  
2. Email to support with written consent  
3. Public posts linking the product (screenshot + link)  
4. Agency/partner case notes with permission  

## Fields required per review

| Field | Required |
|-------|----------|
| Display name (or handle) | yes |
| Context (which tool) | yes |
| Quote (user words) | yes |
| Consent date | yes |
| Source channel | yes |
| Verified by | owner/editor |

## Storage

- Drafts: `docs/content/reviews/drafts/` (not rendered)  
- Published: content registry / CMS only after checklist  
- Homepage testimonials section stays **honest placeholder** until ≥3 verified entries  

## Workflow

1. Collect → log in private notes (no secrets in git)  
2. Confirm consent in writing  
3. Redact PII  
4. Owner OK  
5. Publish  

## Anti-patterns

- Invented names/quotes  
- Stock photos as "users"  
- Star ratings without reviews  

## Exit criteria for trust score uplift

- ≥3 verified published reviews OR section remains non-testimonial  
- `/trust` page stays accurate
