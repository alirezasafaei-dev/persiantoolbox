## JSON چیست و چرا فرمت‌دهی مهم است؟

JSON مخفف **JavaScript Object Notation** است و به عنوان استاندارد مبادله داده بین سرور و کلاینت شناخته می‌شود. تقریباً هر API، فایل پیکربندی و سرویس مدرنی که با آن کار می‌کنید از JSON استفاده می‌کند.

اما JSON خام معمولاً در یک خط طولانی و بدون فاصله نوشته می‌شود. خواندن و اشکال‌زدایی چنین داده‌ای تقریباً غیرممکن است. فرمت‌دهی JSON به شما کمک می‌کند ساختار داده را به صورت خوانا و سلسله‌مراتبی ببینید و سریع‌تر خطاهای احتمالی را شناسایی کنید.

## مشکلات رایج در JSON

برخی از خطاهای رایج در فایل‌های JSON عبارتند از:

- **virgool (ویرگول اضافی):** وجود ویرگول بعد از آخرین آیتم قبل از بسته شدن براکت
- **Trailing commas:** ویرگول انتهایی در آرایه‌ها یا آبجکت‌ها که JSON استاندارد آن را نمی‌پذیرد
- **کلیدهای بدون کوتیشن:** فراموش کردن کوتیشن دور کلیدها
- **comparable mismatch:** عدم تطابق تعداد باز و بسته شدن براکت‌ها و براکت‌های مربعی
- **رشته‌های فاقد escape:** کاراکترهای خاص مثل `"` یا `\` که درون رشته‌ها escape نشده‌اند

این مشکلات می‌توانند باعث خطاهای سخت و غیرقابل تشخیص در برنامه‌های شما شوند.

## استفاده از ابزار فرمت‌دهی JSON

ابزار [فرمت‌دهی JSON PersianToolbox](/tools/json-formatter) چند قابلیت اصلی ارائه می‌دهد:

### فرمت‌دهی زیبا

کد JSON را در textarea وارد کنید و دکمه «فرمت‌دهی» را بزنید. خروجی با تورفتگی مناسب و خطوط جداگانه نمایش داده می‌شود. این ویژگی برای بررسی سریع ساختار API و یافتن خطاهای ساختاری بسیار مفید است.

### فشرده‌سازی

گاهی اوقات نیاز دارید JSON را در یک خط و بدون فاصله اضافی ذخیره کنید — مثلاً برای ارسال در درخواست HTTP یا ذخیره در دیتابیس. با فشرده‌سازی، تمام فاصله‌ها و خطوط اضافی حذف می‌شوند.

### اعتبارسنجی

ابزار به صورت خودکار JSON وارد شده را اعتبارسنجی می‌کند. اگر خطای ساختاری وجود داشته باشد، دقیق شماره خط و نوع خطا را نشان می‌دهد. این کار اشکال‌زدایی را بسیار سریع‌تر می‌کند.

### نمایش درختی

JSON به صورت درختی قابل کلیک و جمع‌شونده نمایش داده می‌شود. برای داده‌های بزرگ و تو‌درتو، این نمایش کمک می‌کند فقط بخش‌های مورد نظر را ببینید و ساختار کلی را سریع درک کنید.

## نکات توسعه

- **همیشه قبل از commit اعتبارسنجی کنید:** حتی اگر از JSONLinter استفاده می‌کنید، یک بار اعتبارسنجی دستی خالی نیست.
- **فشرده‌سازی برای تولید:** در محیط تولید، JSON فشرده حجم کمتری دارد و سرعت انتقال را بالا می‌برد.
- **فرمت‌دهی برای توسعه:** در محیط توسعه، همیشه JSON فرمت‌شده استفاده کنید تا خوانایی کد بالا برود.
- **از escape استفاده کنید:** کاراکترهای خاص مثل `"`، `\` و newlines درون رشته‌ها باید به درستی escape شوند.
- **ابزار ما در مرورگر اجرا می‌شود:** هیچ داده‌ای به سرور ارسال نمی‌شود و حریم خصوصی‌تان حفظ می‌شود.

## سوالات متداول

### آیا ابزار فرمت‌دهی JSON رایگان است؟

بله، کاملاً رایگان و بدون نیاز به ثبت‌نام است.

### آیا اطلاعات من ذخیره می‌شود؟

خیر. تمام عملیات در مرورگر شما انجام می‌شود و هیچ داده‌ای به سرور ارسال نمی‌شود.

### آیا فایل JSON بزرگ پشتیبانی می‌شود؟

بله، ابزار توانایی پردازش فایل‌های JSON بزرگ را دارد.

### خروجی JSON فشرده برای API مناسب است؟

بله. JSON فشرده با حذف فاصله‌ها و خطوط اضافی حجم را کاهش می‌دهد و برای ارسال در API‌ها ایده‌آل است.

### آیا ابزار از JSON5 یا JSONC پشتیبانی می‌کند؟

در حال حاضر ابزار از JSON استاندارد پشتیبانی می‌کند. سینتکس‌هایی مثل trailing commas یا کامنت‌ها معتبر شناخته نمی‌شوند.
