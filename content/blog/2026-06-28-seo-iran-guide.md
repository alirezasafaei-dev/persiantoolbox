## SEO فارسی متفاوت است

بیشتر منابع SEO به زبان انگلیسی نوشته شده‌اند و قوانینی را توضیح می‌دهند که لزوماً برای سایت‌های فارسی صدق نمی‌کنند. سایت‌های فارسی با چالش‌های منحصربه‌فردی روبرو هستند:

- **راست‌چین (RTL):** موتورهای جستجو باید محتوای راست‌چین را به درستی ایندکس کننده
- **کاراکترهای فارسی:** encode شدن صحیح UTF-8 در URL و متا دیتا بسیار مهم است
- **رقابت کمتر:** در حوزه فارسی، رقابت SEO کمتر است و با بهینه‌سازی اصولی می‌توان سریع‌تر به نتایج بالا رسید
- **گوگل فارسی:** گوگل الگوریتم‌های خاصی برای زبان فارسی دارد که باید رعایت شوند

## ساختار URL بهینه

URL نقش مهمی در SEO دارد. برای سایت‌های فارسی:

### اصول طراحی URL

- **از حروف لاتین استفاده کنید:** حتی اگر محتوا فارسی است، URL باید لاتین باشد
- ** slug کوتاه و توصیفی:** مثلاً `salary-calculator` بهتر از `tool-123` است
- **خط تیره به جای زیرخط:** گوگل خط تیره را به عنوان جداکننده کلمات تشخیص می‌دهد
- **بدون کاراکترهای خاص:** از @، #، %، & در URL استفاده نکنید

### مثال‌های درست و غلط

| درست ❌               | غلط ✅                |
| --------------------- | --------------------- |
| `/blog/seo-guide`     | `/blog/راهنمای-seo`   |
| `/tools/salary`       | `/tools/محاسبه-حقوق`  |
| `/blog/check-penalty` | `/blog/check_penalty` |

**نکته مهم:** اگر URL شما حاوی کاراکترهای فارسی باشد، ممکن است در بعضی مرورگرها و شبکه‌های اجتماعی مشکل ایجاد شود. همیشه slug لاتین استفاده کنید.

## متا دیتا و عنوان

### عنوان صفحه (Title Tag)

عنوان صفحه مهم‌ترین عنصر SEO است. ویژگی‌های عنوان خوب:

- **طول مناسب:** بین ۵۰ تا ۶۰ کاراکتر
- **کلمه کلیدی اصلی:** در ابتدای عنوان قرار بگیرد
- **جذاب و توصیفی:** کاربر را ترغیب به کلیک کند
- **نام برند:** در انتهای عنوان با جداکننده

**مثال:**

```html
<title>محاسبه حقوق واقعی — PersianToolbox</title>
<meta
  name="description"
  content="ابزار آنلاین رایگان محاسبه حقوق با احتساب بیمه، مالیات و مزایای ۱۴۰۴"
/>
```

### توضیحات متا (Meta Description)

توضیحات متا تأثیر مستقیم روی نرخ کلیک (CTR) دارد:

- **طول:** بین ۱۵۰ تا ۱۶۰ کاراکتر
- **شامل کلمه کلیدی:** حداقل ۱-۲ بار
- **CTA واضح:** کاربر را به اقدام تشویق کنید
- **اطلاعات مفید:** بگویید ابزار یا مقاله چه کمکی می‌کند

### هدرها (H1, H2, H3)

ساختار هدرها برای خوانایی و SEO مهم است:

- **فقط یک H1:** عنوان اصلی مقاله
- **H2 برای بخش‌های اصلی:** هر بخش مهم یک H2
- **H3 برای زیربخش‌ها:** اطلاعات تکمیلی
- **کلمات کلیدی:** در هدرها به طور طبیعی استفاده شوند

## Schema markup (JSON-LD)

Schema markup به موتورهای جستجو کمک می‌کند محتوای شما را بهتر درک کنند. برای سایت‌های ابزار فارسی، این Schemaها مهم هستند:

### SoftwareApplication

```json
{
  "@context": "https://schema.org",
  "@type": "SoftwareApplication",
  "name": "محاسبه حقوق",
  "applicationCategory": "UtilitiesApplication",
  "operatingSystem": "Web",
  "offers": {
    "@type": "Offer",
    "price": "0",
    "priceCurrency": "IRR"
  }
}
```

### FAQPage

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "آیا ابزار رایگان است؟",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "بله، کاملاً رایگان است."
      }
    }
  ]
}
```

### HowTo

```json
{
  "@context": "https://schema.org",
  "@type": "HowTo",
  "name": "نحوه محاسبه حقوق",
  "step": [
    {
      "@type": "HowToStep",
      "text": "حقوق پایه را وارد کنید"
    }
  ]
}
```

**نکته:** استفاده صحیح از Schema markup می‌تواند Rich Snippets در نتایج گوگل ایجاد کند و نرخ کلیک را افزایش دهد.

## سرعت بارگذاری

سرعت سایت یکی از فاکتورهای رتبه‌بندی گوگل است. نکات مهم:

### بهینه‌سازی تصاویر

- از فرمت WebP استفاده کنید (۳۰٪ کوچک‌تر از PNG)
- تصاویر را قبل از آپلود فشرده کنید
- از Lazy Loading استفاده کنید
- ابعاد تصویر را با نمایشگر تنظیم کنید

### بهینه‌سازی کد

- CSS و JavaScript را Minify کنید
- از Bundle شدن فایل‌ها استفاده کنید
- کدهای غیرضروری را حذف کنید
- از CDN برای فایل‌های استاتیک استفاده کنید

### بهینه‌سازی سرور

- فشرده‌سازی Gzip را فعال کنید
- Browser Caching را تنظیم کنید
- از HTTP/2 یا HTTP/3 استفاده کنید
- زمان پاسخ سرور را زیر ۲۰۰ میلی‌ثانیه نگه دارید

### ابزارهای تست سرعت

- **Google PageSpeed Insights:** تست سرعت و بهینه‌سازی
- **GTmetrix:** تحلیل دقیق عملکرد
- **Lighthouse:** تست جامع عملکرد و SEO

## مثال‌های واقعی

### مثال ۱: ابزار محاسبه حقوق

**قبل از بهینه‌سازی:**

**بعد از بهینه‌سازی:**

### مثال ۲: مقاله راهنما

**قبل از بهینه‌سازی:**

- عنوان: `راهنما`
- توضیحات: `مقاله آموزشی`
- URL: `/blog/post-1`

**بعد از بهینه‌سازی:**

- عنوان: `راهنمای جامع محاسبه عیدی و سنوات — PersianToolbox`

- URL: `/blog/eid-guide`

## سوالات متداول

### آیا SEO برای سایت فارسی با انگلیسی فرق دارد؟

بله. سایت‌های فارسی باید به مسائلی مثل کاراکترهای UTF-8، ساختار URL لاتین، RTL بودن محتوا و الگوریتم‌های خاص گوگل برای فارسی توجه ویژه داشته باشند.

### چقدر طول می‌کشد نتایج SEO را ببینم؟

معمولاً ۳ تا ۶ ماه زمان می‌برد تا تغییرات SEO تأثیر کامل خود را نشان دهد. اما بعضی بهبودها مثل سرعت سایت سریع‌تر تأثیر می‌گذارند.

### آیا Schema markup واقعاً تأثیر دارد؟

بله. Schema markup به گوگل کمک می‌کند محتوا را بهتر درک کند و می‌تواند Rich Snippets ایجاد کند که نرخ کلیک را افزایش می‌دهد.

### بهترین کلمه کلیدی برای سایت ابزار فارسی چیست؟

بستگی به ابزار دارد. برای ابزارهای مالی، عباراتی مثل «محاسبه حقوق»، «ماشین‌حساب مالیات» و «محاسبه وام» پرجستجو هستند.

### آیا استفاده از CDN برای سایت فارسی ضروری است؟

CDN باعث بهبود سرعت بارگذاری برای کاربران ایرانی می‌شود. استفاده از CDN ایرانی مثل آروان یا ابرآروان توصیه می‌شود.

### چگونه بفهمم سایتم چه رتبه‌ای در گوگل دارد؟

از ابزار Google Search Console استفاده کنید. این ابزار رایگان گوگل، اطلاعات دقیقی درباره رتبه صفحات، کلمات کلیدی و عملکرد سایت در جستجو ارائه می‌دهد.
