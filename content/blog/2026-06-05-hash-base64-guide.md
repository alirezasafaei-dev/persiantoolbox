## چرا ابزارهای Hash و Base64 مهم هستند؟

در دنیای دیجیتال امروز، امنیت داده‌ها اولویت اول هر کسب‌وکار و کاربری است. شما روزانه با رمزهای عبور، فایل‌های حساس و اطلاعات شخصی سر و کار دارید. اگر ندانید این اطلاعات چگونه محافظت می‌شوند، در معرض خطرات جدی هستید.

ابزارهای Hash و Base64 دو ستون اصلی امنیت دیجیتال هستند. Hash برای بررسی یکتایی فایل‌ها و ذخیره امن رمزهای عبور استفاده می‌شود و Base64 برای تبدیل داده‌های باینری به متن قابل انتقال. بدون این ابزارها، انتقال ایمیل‌ها، ذخیره رمزهای عبور و حتی بارگذاری تصاویر در وب ممکن نبود.

## Hash چیست؟

Hash (هش) یک تابع ریاضی است که هر ورودی (متن، فایل، تصویر) را به یک رشته خروجی با طول ثابت تبدیل می‌کند. ویژگی کلیدی Hash این است که **غیرقابل برگشت** است: از خروجی نمی‌توان ورودی اصلی را بازیابی کرد.

### ویژگی‌های Hash

| ویژگی         | توضیح                                                |
| ------------- | ---------------------------------------------------- |
| غیرقابل برگشت | از هش نمی‌توان متن اصلی را بازیابی کرد               |
| یکتایی        | هر ورودی دقیقاً یک هش منحصر به فرد دارد              |
| حساس به تغییر | حتی یک کاراکتر تغییر، هش کاملاً متفاوتی تولید می‌کند |
| طول ثابت      | خروجی همیشه طول مشخصی دارد (صرف‌نظر از طول ورودی)    |

### مثال ساده

```text
ورودی: "سلام دنیا"
SHA256: a8f5f167f44f4964e6c998dee827110c97c5b3e4b3f1b2e6e8c9a7d5f4e3c2b1a
```

حتی اگر فقط یک نقطه اضافه کنید ("سلام دنیا.")، هش کاملاً متفاوت خواهد بود. این ویژگی باعث می‌شود Hash برای بررسی صحت فایل‌ها عالی باشد.

## انواع Hash: MD5, SHA1, SHA256

### MD5 (Message Digest 5)

MD5 محبوب‌ترین الگوریتم هش است که خروجی ۱۲۸ بیتی تولید می‌کند. برای بررسی سریع یکتایی فایل‌ها مناسب است اما برای امنیت بالا توصیه نمی‌شود.

| ویژگی     | مقدار                    |
| --------- | ------------------------ |
| طول خروجی | ۳۲ کاراکتر (۱۲۸ بیت)     |
| سرعت      | بسیار بالا               |
| امنیت     | متوسط — شکسته شده        |
| کاربرد    | بررسی صحت فایل، checksum |

### SHA-1 (Secure Hash Algorithm 1)

SHA-1 خروجی ۱۶۰ بیتی تولید می‌کند و امن‌تر از MD5 است. اما گوگل در سال ۲۰۱۷ نشان داد که می‌توان آن را شکست.

| ویژگی     | مقدار                    |
| --------- | ------------------------ |
| طول خروجی | ۴۰ کاراکتر (۱۶۰ بیت)     |
| سرعت      | بالا                     |
| امنیت     | متوسط — شکسته شده        |
| کاربرد    | گواهی‌نامه دیجیتال قدیمی |

### SHA-256 (Secure Hash Algorithm 256-bit)

SHA-256 استاندارد امنیتی فعلی است. در بیت‌کوین، رمزهای عبور و اسناد دیجیتال استفاده می‌شود.

| ویژگی     | مقدار                                  |
| --------- | -------------------------------------- |
| طول خروجی | ۶۴ کاراکتر (۲۵۶ بیت)                   |
| سرعت      | خوب                                    |
| امنیت     | بسیار بالا                             |
| کاربرد    | ذخیره رمز عبور، امضای دیجیتال، بلاکچین |

## Base64 چیست؟

Base64 یک روش رمزنگاری نیست، بلکه یک **روش کدگذاری** است. Base64 داده‌های باینری (مانند تصاویر، فایل‌های صوتی) را به متن ASCII تبدیل می‌کند تا بتوان آن‌ها را در پروتکل‌های متنی مانند HTTP و Email انتقال داد.

### چرا Base64 لازم است؟

پروتکل‌های انتقال مانند HTTP و SMTP فقط متن ساده را پشتیبانی می‌کنند. اگر بخواهید یک تصویر را در ایمیل ضمیمه کنید، باید آن را به Base64 تبدیل کنید.

### مثال عملی

```text
متن اصلی: "سلام دنیا"
Base64:    2LbYlOmHv+e0ouiyoA==
```

### تفاوت Hash و Base64

| ویژگی       | Hash                 | Base64           |
| ----------- | -------------------- | ---------------- |
| هدف         | بررسی یکتایی و امنیت | تبدیل فرمت داده  |
| برگشت‌پذیری | غیرقابل برگشت        | قابل برگشت       |
| کاربرد اصلی | رمز عبور، checksum   | انتقال داده، API |
| حساسیت      | بله                  | خیر              |

## استفاده از ابزارهای جعبه ابزار فارسی

ابزارهای [Hash Generator](/tools/hash-generator) و [Base64 Tool](/tools/base64-tool) جعبه ابزار فارسی کاملاً در مرورگر شما کار می‌کنند. هیچ داده‌ای به سرور ارسال نمی‌شود و حریم خصوصی شما کاملاً حفظ می‌شود.

### Hash Generator

با ابزار Hash Generator می‌توانید:

- متن دلخواه را به MD5، SHA1، SHA256 یا SHA512 تبدیل کنید
- فایل آپلود کنید و هش آن را ببینید
- هش‌های مختلف را مقایسه کنید

### Base64 Tool

ابزار Base64 دو عملیات اصلی انجام می‌دهد:

- ** Encode:** متن ساده را به Base64 تبدیل می‌کند
- **Decode:** Base64 را به متن اصلی برگرداند

## کاربردهای عملی

### ۱. ذخیره امن رمز عبور

هرگز رمز عبور را به صورت ساده ذخیره نکنید. از SHA256 برای هش کردن رمز عبور استفاده کنید:

```text
رمز عبور: MySecretPassword123
SHA256: 5e884898da28047151d0e56f8dc6292773603d0d6aabbdd62a11ef721d1542d8
```

حتی اگر پایگاه داده شما لو برود، رمز عبور واقعی کاربران مشخص نمی‌شود.

### ۲. بررسی صحت فایل‌ها

وقتی یک نرم‌افزار دانلود می‌کنید، سایت ارائه‌دهنده معمولاً هش فایل را هم منتشر می‌کند. شما می‌توانید با ابزار Hash Generator هش فایل دانلود شده را محاسبه و با مقدار اصلی مقایسه کنید. اگر مطابقت نداشت، فایل خراب یا آلوده است.

### ۳. امضای دیجیتال

پیام‌رسان‌ها و ایمیل‌سرویس‌ها برای تأیید صحت پیام‌ها از Hash استفاده می‌کنند. وقتی یک پیام امضا می‌شود، هش پیام با کلید خصوصی فرستنده رمزنگاری می‌شود.

### ۴. انتقال تصاویر در HTML

وقتی می‌خواهید تصویری را مستقیماً در کد HTML قرار دهید (بدون آپلود)، از Base64 استفاده می‌کنید:

```html
<img src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUg..." alt="تصویر" />
```

### ۵. تأیید هویت در API

بسیاری از APIها برای احراز هویت از هش HMAC استفاده می‌کنند. شما درخواست را با یک کلید مخفی هش می‌کنید و سرور همان هش را محاسبه می‌کند تا مطمئن شود درخواست از طرف شماست.

## نکات امنیتی مهم

### از MD5 برای رمز عبور استفاده نکنید

MD5 بسیار سریع شکسته می‌شود. ابزارهای جدید می‌توانند میلیاردها هش MD5 در ثانیه محاسبه کنند. برای رمز عبور حتماً از SHA256 یا بالاتر استفاده کنید.

### هرگز Base64 را رمزنگاری ندانید

Base64 فقط کدگذاری است، نه رمزنگاری. هر کسی می‌تواند آن را به راحتی برگرداند. اگر نیاز به رمزنگاری واقعی دارید، از AES یا RSA استفاده کنید.

### از Salt استفاده کنید

وقتی رمز عبور را هش می‌کنید، یک مقدار تصادفی (Salt) به آن اضافه کنید. این کار باعث می‌شود حتی اگر دو کاربر رمز عبور یکسانی داشته باشند، هش‌هایشان متفاوت باشد.

## سوالات متداول

### آیا ابزار Hash و Base64 رایگان است؟

بله، کاملاً رایگان و بدون نیاز به ثبت‌نام. تمام پردازش‌ها در مرورگر شما انجام می‌شود.

### آیا داده‌های من ذخیره می‌شود؟

خیر. ابزارهای جعبه ابزار فارسی هیچ داده‌ای به سرور ارسال نمی‌کنند. تمام محاسبات محلی هستند.

### تفاوت MD5 و SHA256 چیست؟

MD5 خروجی ۳۲ کاراکتری و SHA256 خروجی ۶۴ کاراکتری تولید می‌کند. SHA256 امن‌تر است و برای رمز عبور و اسناد حساس توصیه می‌شود.

### Base64 رمزنگاری است؟

خیر. Base64 فقط یک روش کدگذاری است. هر کسی می‌تواند آن را به راحتی برگرداند. برای رمزنگاری واقعی از AES یا RSA استفاده کنید.

### آیا می‌توانم فایل را هش کنم؟

بله، ابزار Hash Generator از آپلود فایل پشتیبانی می‌کند. کافی است فایل را بکشید و رها کنید تا هش آن محاسبه شود.

### هش قابل برگشت است؟

خیر. Hash غیرقابل برگشت است. از هش نمی‌توان متن اصلی را بازیابی کرد. این ویژگی باعث امنیت Hash می‌شود.
