# Archive

این پوشه حاوی فایل‌های منسوخ، قدیمی و اضافی است که از دایرکتوری اصلی پروژه حذف شده‌اند تا ریپو تمیز بماند.

## ساختار پوشه

### `legacy-tasks/`

فایل‌های تسک‌های قدیمی که تمام شده‌اند یا جایگزین شده‌اند:

- `alireza_TASK` - لیست تسک‌های شخصی و عملیاتی (تاریخ 2026-02-13)
- `TASKS.md` و `TASKS_NEXT.md` - تسک‌های توسعه قدیمی (تاریخ 2026-02-27)
- `tasks-next/` - فایل‌های تسک NP0, NP1, NP2 که تمام شده‌اند

### `legacy-reports/`

گزارش‌های قدیمی و audit reports:

- `AUDIT_REPORT.md` - گزارش audit و compliance (تاریخ 2026-02-18)

### `legacy-env/`

فایل‌های env قدیمی که جایگزین شده‌اند:

- `env.production.example` - نسخه قدیمی env production
- `env.staging.example` - نسخه قدیمی env staging

### `legacy-licenses/`

فایل‌های لایسنس قدیمی:

- `LICENSE-NONCOMMERCIAL.md` - لایسنس غیرتجاری قدیمی (جایگزین شده با dual-license policy)

### `build-artifacts/`

فایل‌های پیکربندی build قدیمی:

- `lighthouserc.json` - نسخه بایگانی؛ پیکربندی فعال در 2026-06-18 به ریشه پروژه بازگردانده شد چون workflow به آن وابسته است.

### `NOTICE`

فایل notice قدیمی

## چرا این فایل‌ها آرشیو شدند؟

1. **تسک‌های قدیمی:** این تسک‌ها تمام شده‌اند و دیگر مرتبط نیستند
2. **Env files:** نسخه‌های جدیدتر `.env.production.example` وجود دارد
3. **لایسنس‌ها:** سیاست لایسنس جدید با dual-license policy جایگزین شده
4. **گزارش‌ها:** گزارش‌های قدیمی دیگر معتبر نیستند
5. **فایل‌های ساختی:** فایل‌هایی مثل `next-env.d.ts` و `tsconfig.tsbuildinfo` باید با gitignore مدیریت شوند
