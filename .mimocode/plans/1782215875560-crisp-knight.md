Fix the broken monetization foundation, then implement the first premium tool: **Financial Decision Dashboard Pro**.

## Phase A: Foundation Fixes

### A1: Fix `@shared/payments` Module

**Problem:** `lib/payments/payment-integration.ts` imports from `@shared/payments` which doesn't exist on disk.

**Solution:** Create `shared/packages/payments/` with the gateway adapter pattern.

**Files to create:**

- `shared/packages/payments/index.ts` — exports
- `shared/packages/payments/types.ts` — `PaymentGatewayAdapter`, `PaymentConfig`, `PaymentResult`

**Acceptance criteria:**

- Mock adapter works for testing

### A2: Fix Plan ID Mismatch

**Problem:** `subscriptionPlans.ts` uses underscores (`basic_monthly`), `subscription-manager.ts` uses hyphens (`basic-monthly`).

**Solution:** Unify to hyphens (internal standard). Update `subscriptionPlans.ts` to use hyphens.

**Files to modify:**

- `lib/subscriptionPlans.ts` — change plan IDs to hyphens
- `lib/subscriptionPlans.ts` — unify prices with `subscription-manager.ts`

**Acceptance criteria:**

- Plan IDs are consistent across all files
- Prices match between UI and server

### A3: Fix Usage Tracking Premium Detection

**Problem:** `lib/usage-tracking.ts` queries `status = 'ACTIVE'` (uppercase) but subscriptions are stored as `active` (lowercase).

**Solution:** Change the query to use lowercase `active`.

**Files to modify:**

- `lib/usage-tracking.ts` — fix status comparison

**Acceptance criteria:**

- Premium users get unlimited access
- Free users still have 10/day limit

### A4: Add `usage_tracking` Table

**Problem:** `usage-tracking.ts` uses a `usage_tracking` table that's not in `schema.sql`.

**Solution:** Add the table definition.

**Files to modify:**

## Phase B: Financial Decision Dashboard Pro

### B1: Database Schema

Add to `schema.sql`:

```sql
CREATE TABLE IF NOT EXISTS financial_scenarios (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  title text NOT NULL,
  scenario_type text NOT NULL,
  inputs jsonb NOT NULL,
  outputs jsonb,
  notes text DEFAULT '',
  created_at bigint NOT NULL,
  updated_at bigint NOT NULL
);
CREATE INDEX IF NOT EXISTS financial_scenarios_user_idx ON financial_scenarios (user_id);
```
